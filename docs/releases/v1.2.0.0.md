# v1.2.0.0

## What changed

### Interaction & Usability (P0 critical fixes)
- **Double-click to transfer** — Double-clicking a file in the local pane (when connected) uploads it to the current remote directory; double-clicking a file in the remote pane downloads it to the current local directory. Enter key triggers the same action on the focused item.
- **Keyboard shortcuts in file panes** — F5 refreshes the listing, Backspace navigates up one directory level, Enter opens a directory or activates a file. Shortcuts are scoped to the pane container and do not fire when the user is typing in an input field.
- **Drag-and-drop visual feedback** — File panes now display a dashed blue outline and faint blue tint when a valid drag is hovering over them. The drop zone clears immediately on drag leave or drop.
- **In-app delete confirmation** — The delete action on remote items no longer uses `window.confirm()` (a blocking, off-theme browser dialog). A dark-themed `ConfirmDialog` overlay is rendered instead, with Cancel and Delete buttons, backdrop-click to dismiss, and clear wording including a directory recursion warning.
- **Context menu keyboard navigation** — Arrow Up/Down, Home, End, Tab, and Enter all work within open context menus. Mouse hover and keyboard focus are kept in sync. The focused item scrolls into view. The menu auto-focuses on open so keyboard events are captured immediately. ARIA `role="menu"` and `role="menuitem"` attributes are applied.

### Transfer Queue UX
- **Per-row dismiss button** — Each transfer row shows a hover-only ✕ button to remove it from the queue individually.
- **Clear completed** — A "Clear completed" toolbar button appears above the list whenever any transfers have a terminal status (complete or error), removing them all in one click.
- **Status color coding** — Transfer rows are tinted by status: amber for pending, blue for active, green for complete, red for error. The pending clock icon is amber to match.

### Security Dialog UX
- **Revised Trust/Reject button hierarchy** — In both SSH host-key and FTPS certificate prompts, Reject is now amber (warning color, left/first) and Trust and Continue is muted gray with an outline (right, deliberate secondary choice). Neither button uses blue, which previously implied a safe "proceed" action on the riskier option.
- **Messages tab amber badge** — When a security prompt arrives, the Messages tab badge turns amber with a pulse animation instead of the standard blue, drawing attention without disrupting the active tab.

### Visual & Design System
- **Extension-based file icons** — 40+ file types now render distinct icons and colors: images (cyan), archives (purple), code files (blue/yellow/green by language), video (pink), database (orange). Folder icons are unchanged (yellow).
- **Splitter redesign** — The pane splitter hit target is widened from 4 px to 8 px. A centered 2 px visual grip is rendered via a CSS `::after` pseudo-element; on hover/drag it widens to 3 px and turns blue.
- **File row sizing** — Row height increased from 22 px to 24 px; horizontal padding from 4 px to 6 px for better scannability.
- **Progress bar** — Height increased from 4 px to 6 px. Animation tightened from 300 ms to 150 ms `ease-out` with `will-change: width` for GPU-accelerated transitions.

## Why

A 6-agent parallel UI/UX audit of the application identified 59 distinct issues across layout, interaction, file listing, transfer feedback, menus, overlays, typography, and workflow efficiency. An independent adversarial reviewer consolidated the findings into a prioritized backlog of P0 (critical usability blockers) and P1 (high-impact improvements). A separate independent critic reviewed the implementation plan and required revisions before code was written. This release addresses all P0 and the highest-impact P1 items.

## Migration steps

None — all changes are additive UI behavior. The C# backend and bridge API are unchanged.

## Breaking changes

None.

## Known caveats

- **Transfer cancel not yet implemented** — Cancelling an in-progress transfer requires a new `CancelTransfer` bridge method on the C# side. The dismiss button on active transfers is hidden for now; only completed and failed transfers can be dismissed.
- **Breadcrumb path navigation deferred** — Clickable breadcrumb segments in the path bar were planned but deferred due to Windows path parsing edge cases (drive letters, UNC paths, root navigation). The path bar remains a static read-only display for this release.
- **Multi-select deferred** — Ctrl+Click / Shift+Click multi-selection requires react-arborist API investigation and was deferred to a follow-up release.
- **TopBar layout reorganization deferred** — The QuickConnect form grouping and visual hierarchy improvements to the top bar are planned for a subsequent release.
